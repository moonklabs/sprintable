# 진행자 가이드 — AI가 만든 일을 어떻게 믿고 받아들일까

## 세션 개요

- 권장 시간: 20~30분
- 대상: 기획자·디자이너·대표·팀 리드·개발자 혼합 그룹
- 준비물: `index.html`, 참가자가 최근 AI와 함께 처리한 실제 task 1건
- 목표: 제품 설명을 듣는 세션이 아니라, 참가자가 자신의 업무에서 `수용 병목`을 찾아내는 세션

## 20분 진행안

### 0~3분 — 장면으로 시작

다음 질문을 먼저 던진다.

> AI가 하루 만에 시안·문서·코드를 10개 만들어 왔다면, 누가 어떤 기준으로 그중 하나를 고객에게 내보낼까요?

제품이나 기술 용어를 먼저 설명하지 않는다. 참가자가 최근 겪은 `AI가 다 했다고 했지만 내가 다시 확인한 장면`을 떠올리게 한다.

### 3~7분 — 병목 이름 붙이기

핵심 문장:

> 만드는 속도는 병렬로 늘었지만, 책임 있는 판단은 여전히 사람 한 명 앞에 직렬로 선다.

`컨텍스트 스위칭`, `AI가 말을 안 듣는다`, `도구가 너무 많다`는 독립 문제라기보다 수용을 위해 사람이 여러 곳을 확인하는 과정에서 생기는 증상이라고 설명한다.

### 7~11분 — 네 도구의 역할

- Linear: 무엇을 왜 할지, 누가 책임질지
- Claude/Codex: 실제 결과물을 만드는 세션
- GitHub: 실제 코드 변경과 CI·리뷰
- Sprintable: 완료 주장, 증거, 사람 판정, 거절 이유, 이후 결과

강조할 점:

> Sprintable이 모든 데이터를 복사해 소유하는 것이 아니다. 각 도구의 정본을 연결해 수용 결정을 재현 가능하게 만든다.

### 11~15분 — Advisor 설명

Advisor를 `또 다른 승인자`라고 설명하지 않는다.

> 담당자가 결재를 올리기 전에 빠진 첨부파일·숫자 오류·과거 반려 사유를 먼저 확인해 주는 선임 검토자다.

Advisor 결과는 네 가지로 제한한다.

- READY — 사람에게 올릴 준비가 됨
- FIX FIRST — 사람에게 오기 전에 고칠 수 있음
- ASK HUMAN — 목표·정책·위험 판단이 필요함
- UNVERIFIED — 증거가 부족해 판단할 수 없음

최종 승인 책임은 사람에게 남는다.

### 15~20분 — 참가자 업무에 적용

`workshop-sheet.md`에서 다음 다섯 칸만 채운다.

1. 우리가 원하는 결과
2. 완료라고 말할 수 있는 증거
3. AI가 사람 전에 고칠 수 있는 것
4. 사람만 판단할 수 있는 것
5. 되돌릴 수 없어 승인해야 하는 순간

## 직군별 질문

### 기획자에게

- 이 task가 완료됐다고 말하려면 어떤 사용자 상태까지 확인해야 하나?
- 요구사항이 애매할 때 AI가 묻고 시작해야 하는 질문은 무엇인가?

### 디자이너에게

- 정상 화면 외에 empty/error/loading/mobile/accessibility 상태는 포함됐나?
- 결과물이 브랜드 원칙을 어겼다고 판단할 근거는 어디에 남아 있나?

### 대표·팀 리드에게

- 반드시 본인이 도장을 찍는 사건은 merge, deploy, 고객 발송, 결제 중 무엇인가?
- 현재는 그 결정을 내리기 위해 몇 개의 화면과 사람을 확인하는가?

### 개발자에게

- 테스트가 통과했다는 주장과 실제 실행된 command/SHA가 연결돼 있는가?
- reject 이유가 다음 agent 세션에 구조적으로 돌아가는가?

## 흔한 오해와 교정

| 오해 | 교정 문장 |
|---|---|
| AI를 감시하는 도구다 | 감시가 아니라 사람이 판단할 증거를 먼저 정리하는 도구다. |
| AI가 자동 승인한다 | Advisor는 선제 검토자이고 최종 책임·승인은 사람에게 남는다. |
| 모든 일에 승인이 필요하다 | 되돌리기 어려운 일과 정책 판단에만 사람 주의력을 쓴다. |
| Linear를 대체한다 | Linear의 계획·소유권을 그대로 쓰고 완료 증명과 판정 기억만 보완한다. |
| 대화를 전부 저장해야 한다 | 전체 채팅보다 task, SHA, 증거, 판정, 결과를 연결한 Work Manifest가 중요하다. |

## 세션 종료 문장

참가자에게 아래 문장을 자신의 업무 예시로 완성하게 한다.

> 우리 팀에서 AI가 만드는 것은 ______이고, 사람이 반드시 받아들여야 하는 순간은 ______이다. Sprintable은 그 전에 ______을 확인해 준다.
